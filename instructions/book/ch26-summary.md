**Книга:** «Data Science с нуля» — Джоэл Грасс (1-е изд., 2017)
**Источник:** PDF (1-е издание, 2017) — нет файла `scratch/`

> **Контекст:** глава, которой нет во 2-м издании как отдельной (часть тем вошла в общий апдейт 2019). Поднимает вопросы **конфиденциальности**, **согласия**, **bias** в данных и ответственности дата-сайентиста за последствия моделей. Главный тезис: «технически возможно» ≠ «этически допустимо».

## Ключевые концепции

- **Информированное согласие** — данные пользователей не могут быть «собраны по умолчанию»; GDPR, CCPA и аналоги обязывают явно получать разрешение
- **Анонимизация ≠ приватность** — re-identification по комбинации признаков (zip + DOB + пол) тривиальна; нужны дифференциальная приватность или k-anonymity
- **Algorithmic bias** — модель учит **историческое неравенство** из данных (COMPAS, hiring tools); «garbage in, garbage out» приобретает социальный смысл
- **Disparate impact** — формально одинаковые правила дают разный эффект для разных групп (тест «4/5 rule» в США)
- **Right to explanation** — GDPR даёт право на объяснение автоматического решения; касается и ML-моделей
- **Data brokers** — посредники между источниками и потребителями данных; «теневая экономика»
- **Reproducibility crisis** — модели не пересобираются, датасеты утекают, результаты не воспроизводятся; нужны version control для данных и кода
- **«Do no harm»** — главный принцип; ML-система может причинить вред даже без злого умысла

```python
# Чек-лист перед деплоем модели
checks = {
    "consent":                "есть ли явное согласие на сбор данных?",
    "fairness":               "проверены ли метрики по защищённым группам?",
    "explainability":         "можем ли объяснить решение конкретному пользователю?",
    "data_lineage":           "известно ли происхождение каждого признака?",
    "retention_policy":       "когда и как удаляются PII?",
    "audit_log":              "логируются ли все предсказания для разбора инцидентов?",
    "human_in_the_loop":      "есть ли финальное решение человеком в критичных доменах?",
}
```

## Краткие выводы

1. **Bias — техническая, а не только социальная проблема** — модель воспроизводит данные, а данные отражают общество
2. **Анонимизация не спасает** — linkage-атаки восстанавливают личность
3. **Этика ≠ юридический compliance** — это про доверие пользователей и долгосрочную репутацию
