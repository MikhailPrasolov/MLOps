## Ключевые концепции

- **«Не изобретай велосипед»** — для production берите `scikit-learn`, `pandas`, `TensorFlow`/`PyTorch`
- **NumPy** — векторные вычисления; `np.array` быстрее Python-списков в 100× на больших данных
- **pandas** — `DataFrame` для табличных данных; `groupby`, `merge`, `pivot` — то, что мы делали вручную
- **scikit-learn** — десятки алгоритмов «из коробки»; глава 14-20 ≈ его API
- **Глубокое обучение** — TensorFlow / PyTorch; главы 18-19 вводят, библиотеки дают GPU-ускорение
- **Статистика и R** — если нужны «продвинутые» тесты, модели, визуализации
- **Распределённые вычисления** — Spark / Dask; масштабирование за пределы одной машины
- **Domain expertise** — книга даёт инструменты; **смысл** приходит из понимания предметной области

## Код

В главе нет кода — это roadmap. Практический «следующий шаг» в коде:

```python
# Маршрут после книги: заменить scratch на production-стек
# pip install numpy pandas scikit-learn torch

import numpy as np
import pandas as pd
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import train_test_split

# Замена главы 16 в одну строку
df = pd.read_csv("data.csv")
X_train, X_test, y_train, y_test = train_test_split(
    df.drop("target", axis=1), df["target"], test_size=0.25
)
model = LogisticRegression().fit(X_train, y_train)
print(f"accuracy = {model.score(X_test, y_test):.3f}")
```
